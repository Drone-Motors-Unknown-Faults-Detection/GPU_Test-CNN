# MNIST — 手寫數字辨識

## 概述

MNIST 是最入門的影像分類基準，包含 0 ~ 9 共 10 種手寫數字的灰階圖片。本模組以雙層 CNN 實作，訓練約 53 秒（Apple M4 Pro）即可達到 99% 以上測試準確率。

- **腳本**：`src/mnist.py`
- **執行腳本**：`mnist.sh`
- **日誌目錄**：`logs/MNIST/`
- **模型輸出**：`mnist_cnn.pth`

---

## 資料集

| 項目 | 內容 |
|------|------|
| 來源 | `torchvision.datasets.MNIST`（自動下載至 `./data/`） |
| 訓練集 | 60,000 張 |
| 測試集 | 10,000 張 |
| 圖片大小 | 28 × 28（灰階，1 通道） |
| 類別數 | 10（數字 0 ~ 9） |

### 前處理

```python
transforms.ToTensor()
transforms.Normalize(mean=(0.1307,), std=(0.3081,))
```

訓練集與測試集套用相同 transform，不做額外增強（資料集夠簡單，增強無顯著效益）。

---

## 模型架構

```
輸入 (1 × 28 × 28)
│
├─ Conv2d(1→32, kernel=5, padding=2) + ReLU + MaxPool(2,2)
│    輸出：32 × 14 × 14
│
├─ Conv2d(32→64, kernel=5, padding=2) + ReLU + MaxPool(2,2)
│    輸出：64 × 7 × 7
│
├─ Flatten  →  3136
├─ Linear(3136 → 128) + ReLU + Dropout(0.5)
└─ Linear(128 → 10)
     輸出：10 個 logits
```

### 設計重點

- **5×5 卷積核**：相比 3×3 具有更大感受野，適合辨識筆劃形狀。
- **MaxPool**：每層後縮減一半空間尺寸，減少參數量。
- **Dropout(0.5)**：全連接層前加 dropout，防止對訓練集過擬合。
