# CNN 模型運作簡介

## Epoch（訓練輪次）

一個 epoch 代表模型把**整個訓練資料集看過一遍**。

MNIST 有 60,000 張圖，batch size 設為 128，所以一個 epoch 會跑 469 個 batch。
訓練 10 個 epoch，就等於整個資料集被看過 10 次。

重複看是因為一次看完通常學得不夠，多看幾遍模型才會越來越準。

---

## 卷積（Convolution）

卷積是 CNN 的核心，負責**從圖片中提取特徵**。

做法是用一個小的過濾器（filter / kernel）在圖片上滑動，每次計算過濾器與對應區域的點積：

```
圖片區域        過濾器          輸出
1 0 1           1 0            ...
0 1 0     ×     0 1      =    一個數字
1 0 1           1 0
```

不同的過濾器會抓到不同的特徵，例如：
- 邊緣、輪廓
- 曲線、角落
- 更深層的抽象形狀

這個程式的第一層卷積用了 32 個過濾器，第二層用了 64 個：

```python
self.conv1 = nn.Conv2d(1, 32, kernel_size=5, padding=2)
self.conv2 = nn.Conv2d(32, 64, kernel_size=5, padding=2)
```

---

## Pooling（池化）

卷積後的特徵圖解析度很高，Pooling 負責**縮小尺寸、保留重要資訊**。

這個程式用的是 Max Pooling，在每個 2×2 的區塊中只保留最大值：

```
4 2      Max Pooling      4
1 3      ──────────►
```

效果是把 28×28 的圖逐步縮小：
- 卷積後 → 28×28
- 第一次 Pooling → 14×14
- 第二次 Pooling → 7×7

好處：
- 減少計算量
- 對位移、微小形變更有容忍度

---

## CNN 整體流程

```
輸入圖片 (1×28×28)
    │
    ▼
卷積層 1 + ReLU  →  32 個特徵圖 (32×28×28)
    │
    ▼
Max Pooling      →  縮小一半 (32×14×14)
    │
    ▼
卷積層 2 + ReLU  →  64 個特徵圖 (64×14×14)
    │
    ▼
Max Pooling      →  縮小一半 (64×7×7)
    │
    ▼
攤平 (Flatten)   →  3136 個數字
    │
    ▼
全連接層 + ReLU  →  128 個神經元
    │
    ▼
Dropout          →  隨機關掉 50% 神經元（防止過擬合）
    │
    ▼
輸出層           →  10 個數字（對應 0~9 的機率）
```

---

## ReLU 激活函數

每次卷積後都接一個 ReLU，作用是**把負值歸零**：

```
ReLU(x) = max(0, x)
```

沒有激活函數的話，不管疊多少層都只是線性運算，模型學不到複雜的特徵。

---

## Dropout

訓練時隨機讓 50% 的神經元不作用，讓模型不能依賴特定神經元，強迫它學到更通用的特徵，避免**過擬合**（在訓練集準確但測試集差）。

測試時 Dropout 自動關閉，所有神經元都會參與運算。

## 評估指標

### Loss（損失）

Loss 衡量**模型預測與正確答案之間的差距**，數字越小代表模型越準。

這個程式用的是 Cross-Entropy Loss，適合分類問題：

```
Loss = -log(模型給出正確類別的機率)
```

例如圖片是數字 3，模型給出 3 的機率是 0.9，Loss ≈ 0.10；
若機率只有 0.1，Loss ≈ 2.30。

訓練時透過反向傳播不斷調整權重讓 Loss 下降。

---

### Accuracy（準確率）

最直觀的指標，代表**預測正確的比例**：

```
Accuracy = 預測正確筆數 / 總筆數
```

例如 10,000 張測試圖預測對 9,850 張，Accuracy = 98.5%。

MNIST 是十個類別分佈均勻的資料集，Accuracy 在這裡夠用。
但若資料不均衡（例如 99% 都是同一類），Accuracy 就會誤導判斷。

---

### Confusion Matrix（混淆矩陣）

在介紹 Precision / Recall / F1 之前，先理解四種預測結果：

以「是否為數字 7」為例（二元分類）：

|                  | 實際是 7        | 實際不是 7      |
|------------------|-----------------|-----------------|
| **預測是 7**     | TP（真陽性）    | FP（假陽性）    |
| **預測不是 7**   | FN（假陰性）    | TN（真陰性）    |

- **TP**：預測是 7，實際也是 7（答對）
- **FP**：預測是 7，實際不是 7（誤報）
- **FN**：預測不是 7，實際是 7（漏報）
- **TN**：預測不是 7，實際也不是 7（答對）

---

### Precision（精確率）

在**所有預測為正的樣本中**，實際為正的比例：

```
Precision = TP / (TP + FP)
```

衡量「模型說是，有多少比例真的是」。  
Precision 低代表誤報多，例如把很多非 7 都判成 7。

---

### Recall（召回率）

在**所有實際為正的樣本中**，模型成功找出的比例：

```
Recall = TP / (TP + FN)
```

衡量「真正的正樣本，模型抓到了多少」。  
Recall 低代表漏報多，例如很多 7 都被判成其他數字。

---

### Precision 與 Recall 的取捨

兩者通常是反向關係：

- 判斷標準**放寬** → 更多東西被預測為正 → Recall 上升，Precision 下降
- 判斷標準**收緊** → 只有很確定才預測為正 → Precision 上升，Recall 下降

實際應用中要根據需求決定哪個更重要，例如：
- 醫療診斷：Recall 優先（寧可誤報，不能漏掉病人）
- 垃圾郵件過濾：Precision 優先（寧可漏掉，不能誤刪正常信）

---

### F1 Score

F1 是 Precision 和 Recall 的**調和平均數**，當兩者都重要時用這個：

```
F1 = 2 × (Precision × Recall) / (Precision + Recall)
```

範圍 0 到 1，越高越好。  
只有當 Precision 和 Recall 都高時，F1 才會高；任一方很低都會拉低整體分數。

---

### 各指標總結

| 指標        | 衡量什麼                         | 越高越好 |
|-------------|----------------------------------|----------|
| Loss        | 預測與答案的差距                 | 越低越好 |
| Accuracy    | 整體預測正確比例                 | ✓        |
| Precision   | 預測為正時有多準                 | ✓        |
| Recall      | 實際為正時抓到多少               | ✓        |
| F1 Score    | Precision 與 Recall 的綜合評分   | ✓        |
